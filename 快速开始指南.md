# 🚀 快速开始指南

## 第一步：环境准备

### 1.1 系统要求
- Ubuntu 22.04+ / macOS 12+ / Windows 10+ (WSL2推荐)
- Python 3.12+ 
- Docker 20.10+
- Docker Compose 2.0+

### 1.2 安装依赖

**Ubuntu/Debian:**
```bash
# 安装系统依赖
sudo apt update
sudo apt install -y python3 python3-pip python3-venv docker.io docker-compose git

# 添加用户到docker组（需要重新登录）
sudo usermod -aG docker $USER
```

**macOS:**
```bash
# 使用Homebrew
brew install python@3.11 docker docker-compose git
```

**Windows (WSL2):**
```bash
# 在WSL2中
sudo apt update
sudo apt install -y python3 python3-pip python3-venv

# 从Windows安装Docker Desktop for WSL2
```

## 第二步：设置开发环境

### 2.1 克隆项目（如果尚未克隆）
```bash
git clone <项目地址>
cd ai-trading-system
```

### 2.2 创建虚拟环境
```bash
python3 -m venv venv
source venv/bin/activate  # Linux/macOS
# 或 venv\Scripts\activate  # Windows
```

### 2.3 安装Python依赖
```bash
# 升级pip
pip install --upgrade pip

# 安装基础依赖
pip install -r requirements.txt

# 安装开发依赖
pip install -r requirements-dev.txt

# 安装测试依赖
pip install -r requirements-test.txt

# 或一次性安装所有
pip install -e ".[dev,test,prod]"
```

### 2.4 配置环境变量
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置文件
# 使用你喜欢的编辑器编辑 .env 文件
nano .env  # 或 vim, code, 等
```

**重要配置项：**
```bash
# 数据库配置
DATABASE_URL=postgresql://trader:changeme123@localhost:5432/trading_db

# Redis配置
REDIS_URL=redis://localhost:6379/0

# 安全配置
SECRET_KEY=your-very-secret-key-change-this-in-production
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30

# 监控配置
PROMETHEUS_PORT=9091
GRAFANA_PORT=3000
```

## 第三步：启动基础设施

### 3.1 启动Docker服务
```bash
# 启动必要的基础设施
docker-compose up -d postgres redis

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 3.2 初始化数据库
```bash
# 运行数据库迁移
alembic upgrade head

# 或使用脚本
python scripts/init_db.py
```

### 3.3 验证基础设施
```bash
# 检查数据库连接
docker-compose exec postgres pg_isready -U trader

# 检查Redis连接
docker-compose exec redis redis-cli ping

# 检查所有服务健康
./scripts/health-check.sh
```

## 第四步：运行开发服务器

### 4.1 启动开发服务器
```bash
# 方式1：直接运行
python -m src.main

# 方式2：使用uvicorn（支持热重载）
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000

# 方式3：使用Makefile
make run
```

### 4.2 验证应用运行
```bash
# 检查应用健康
curl http://localhost:8000/health

# 检查API文档
curl http://localhost:8000/docs  # OpenAPI文档
curl http://localhost:8000/redoc  # ReDoc文档
```

## 第五步：运行测试

### 5.1 运行所有测试
```bash
# 运行所有测试
pytest

# 运行单元测试
pytest -m unit

# 运行集成测试
pytest -m integration

# 运行E2E测试
pytest -m e2e
```

### 5.2 生成测试报告
```bash
# 生成覆盖率报告
pytest --cov=src --cov-report=html

# 打开覆盖率报告
open htmlcov/index.html  # macOS
xdg-open htmlcov/index.html  # Linux
start htmlcov/index.html  # Windows
```

### 5.3 代码质量检查
```bash
# 代码格式化
black src tests
isort src tests

# 代码检查
flake8 src tests
mypy src

# 安全检查
safety check
bandit -r src
```

## 第六步：开发工作流程

### 6.1 创建新功能分支
```bash
# 从develop分支创建新功能分支
git checkout develop
git pull origin develop
git checkout -b feature/your-feature-name
```

### 6.2 开发新功能
1. 编写代码
2. 编写测试
3. 运行测试
4. 代码检查

### 6.3 提交代码
```bash
# 添加文件
git add .

# 提交（使用Conventional Commits格式）
git commit -m "feat(module): add new feature"

# 或使用更详细的提交信息
git commit -m "feat(module): add new feature

- 实现XXX功能
- 添加相关测试
- 更新文档

Closes #123"
```

### 6.4 创建Pull Request
1. 推送到远程分支
2. 在GitHub/GitLab创建PR
3. 等待代码审查
4. 通过CI检查
5. 合并到develop分支

## 常见问题解决

### 问题1：数据库连接失败
```bash
# 检查数据库服务
docker-compose ps postgres

# 检查端口占用
netstat -tulpn | grep :5432

# 重启数据库服务
docker-compose restart postgres
```

### 问题2：Redis连接失败
```bash
# 检查Redis服务
docker-compose ps redis

# 检查Redis日志
docker-compose logs redis

# 清除Redis数据
docker-compose exec redis redis-cli FLUSHALL
```

### 问题3：Python依赖问题
```bash
# 重建虚拟环境
deactivate
rm -rf venv
python3 -m venv venv
source venv/bin/activate
pip install -e ".[dev,test,prod]"
```

### 问题4：Docker容器问题
```bash
# 清理所有容器和卷
docker-compose down -v
docker system prune -f

# 重新启动
docker-compose up -d --build
```
