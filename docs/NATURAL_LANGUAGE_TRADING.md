# 🎯 自然语言指令驱动交易系统

## 📖 概述

本系统支持**完全自然语言驱动**的量化交易！用户可以用普通中文对话的方式下达指令，系统自动理解并执行完整的交易流程。

---

## 🚀 完整的智能流程

```
用户在前端说："帮我开仓BTC多单"
                ↓
        前端AI对话窗口
                ↓
        /api/v1/ai/chat 端点
                ↓
    AI指令执行器 (ai_command_executor.py)
                ↓
    AI指令解析器 (ai_command_parser.py)
                ↓
    解析为：OPEN_POSITION, BTC/USDT, LONG
                ↓
    ┌─────────────┴─────────────┐
    ↓                           ↓
是交易指令？              是普通对话？
    ↓                           ↓
执行对应操作             LLM直接回答
（采集数据→分析→决策→执行）   （带记忆）
    ↓                           ↓
返回结果给前端           返回结果给前端
```

## 🧠 核心组件

### 1. AI指令解析器 (`ai_command_parser.py`)

**功能：**
- 将自然语言解析为结构化指令
- 支持基于规则的快速解析
- 支持AI智能解析
- 支持多轮对话上下文理解

**支持的指令类型：**
```python
class CommandType(Enum):
    OPEN_POSITION = "open_position"       # 开仓
    CLOSE_POSITION = "close_position"     # 平仓
    ANALYZE_MARKET = "analyze_market"     # 分析市场
    START_TRADING = "start_trading"       # 开始自动交易
    STOP_TRADING = "stop_trading"         # 停止自动交易
    QUERY_STATUS = "query_status"         # 查询状态
    CHAT = "chat"                         # 普通对话
```

---

### 2. AI指令执行器 (`ai_command_executor.py`)

**功能：**
- 接收解析后的指令
- 执行对应的操作
- 协调交易引擎、业务流程等组件
- 后台任务管理
- 记忆保存

**处理流程：**
```
用户输入 → 解析指令 → 执行对应操作 → 返回结果
                        ↓
                启动后台任务
                        ↓
                执行完整流程
                        ↓
                记录到记忆
```

---

### 3. 完整集成流程

```
前端对话窗口
    ↓
/api/v1/ai/chat (server.py)
    ↓
AI指令执行器 (ai_command_executor.py)
    ↓
AI指令解析器 (ai_command_parser.py)
    ↓
根据指令类型分发：
    ├─→ 开仓 → 启动开仓流程 → AITradingEngine
    ├─→ 平仓 → 启动平仓流程 → AITradingEngine
    ├─→ 分析市场 → 启动分析流程 → BusinessProcessManager
    ├─→ 开始交易 → 启动AITradingEngine
    ├─→ 停止交易 → 停止AITradingEngine
    ├─→ 查询状态 → 获取系统状态
    └─→ 普通对话 → LLMIntegration (带记忆)
    ↓
返回结果给前端
```
