## 配置管理

### 文件位置
- **主文件**: `src/modules/core/config_manager.py`

### 功能特性

#### 1. 配置加载
- ✅ 支持 JSON 格式配置文件
- ✅ 支持 YAML 格式配置文件（可选依赖）
- ✅ 支持环境变量配置（前缀：`TRADING_`）
- ✅ 自动检测配置文件变化并热重载

#### 2. 配置验证
- ✅ 支持 Pydantic 模型验证（可选依赖）
- ✅ 简化验证（无依赖时自动降级）
- ✅ 配置事务支持（批量更新）

#### 3. 配置监听
- ✅ 支持配置变更监听
- ✅ 异步回调机制
- ✅ 配置变更历史记录

### 使用示例

```python
from src.modules.core.config_manager import ConfigManager

# 初始化
config = ConfigManager(config_dir="config")
await config.initialize()

# 获取配置
value = await config.get_config("section", "key", default="default_value")

# 设置配置
await config.set_config("section", "key", "value")

# 监听配置变更
async def on_config_change(section, key, old_value, new_value):
    print(f"配置变更: {section}.{key}")

await config.watch_config("section", "key", on_config_change)

# 事务操作
async with config.transaction() as tx:
    await tx.set_config("db", "host", "localhost")
    await tx.set_config("db", "port", 5432)
```

### ⚠️ 注意事项
- **不要**创建新的配置管理器
- **不要**重复实现配置加载功能
- **不要**重复实现配置热重载功能
- 如需扩展，请在现有基础上添加功能

## 开发规范

### 1. 开发前检查清单

在开发新功能前，请确认：

- [ ] 查阅本指南，确认功能是否已存在
- [ ] 检查相关模块的源代码
- [ ] 确认是否可以通过扩展现有功能实现
- [ ] 避免重复造轮子

### 2. 扩展现有功能

如需扩展功能，请遵循以下原则：

```python
# ✅ 正确做法：扩展现有类
from src.modules.core.strategy_manager import BaseStrategy

class MyNewStrategy(BaseStrategy):
    """新策略继承基类"""
    async def generate_signals(self, data):
        # 实现新策略逻辑
        pass

# ❌ 错误做法：重复实现
class MyStrategyManager:  # 不要创建新的管理器
    """重复实现策略管理"""
    pass
```

### 3. 代码审查要点

- 是否重复实现了已有功能？
- 是否可以通过扩展现有类实现？
- 是否遵循了单一职责原则？
- 是否添加了适当的文档和注释？

### 4. 文档更新

添加新功能后，请更新：

- [ ] 本功能缩影指南

## 常见问题

### Q: 如何确认功能是否已存在？
A: 
1. 查阅本指南
2. 搜索相关关键词：`grep -r "功能名" src/`
3. 查看模块源代码

### Q: 发现功能缺失怎么办？
A:
1. 确认是否真的缺失（检查所有相关模块）
2. 如确实缺失，在现有模块基础上扩展
3. 更新本指南

### Q: 如何扩展现有功能？
A:
1. 继承现有基类
2. 重写或添加新方法
3. 保持向后兼容
4. 添加单元测试
